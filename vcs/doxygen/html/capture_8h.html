Detailed Description

The interface to VCS's capture subsystem.

The capture subsystem is responsible for mediating exchanges between VCS and the capture device; including initializing the capture device and providing VCS with access to the frame buffer(s) associated with the device.

Usage:

  1. Call kc_initialize_capture() to initialize the subsystem. This is VCS's default startup behavior.
  2. Use the interface returned by kc_capture_device() to interact with the capture device.
  3. Call kc_release_capture() to release the subsystem. This is VCS's default exit behavior.

Enumeration Type Documentation

◆ capture_deinterlacing_mode_e

enum capture_deinterlacing_mode_e
strong

Enumerates the de-interlacing modes recognized by the capture subsystem.

Note
The capture subsystem itself doesn't apply de-interlacing, it just asks the capture device to do so. The capture device in turn may support only some or none of these modes, and/or might apply them only when receiving an interlaced signal.

◆ capture_event_e

enum capture_event_e
strong

VCS will periodically query the capture subsystem for the latest capture events. This enumerates the range of capture events that the capture subsystem can report back.

Enumerator
none

No capture events to report. Although some capture events may have occurred, the capture subsystem chooses to not inform VCS of them.

sleep

Same as 'none', but the capture subsystem also thinks VCS shouldn't send a new query for capture events for some short while (milliseconds); e.g. because the capture device is currently not receiving a signal and so isn't expected to produce events in the immediate future.

signal_lost

The capture device has just lost its input signal.

new_frame

The capture device has sent a new frame, whose data is now available tp VCS via the capture device interface.

new_video_mode

The capture device's input signal has changed in resolution or refresh rate. The new mode parameters can be queried via the capture device interface.

invalid_signal

The capture device's current input signal is invalid (e.g. out of range).

invalid_device

The capture device isn't available for use.

unrecoverable_error

An error has occurred within the capture subsystem from which the subsystem can't recover.

num_enumerators

Total enumerator count. Should remain the last item in the list.

◆ capture_pixel_format_e

enum capture_pixel_format_e
strong

Enumerates the color formats recognized by the capture subsystem. Frames output by the subsystem will have their pixel data in one of these formats.

The capture device may support some, none, or all of these formats. Any capability queries should be made via the capture device interface.

See also
captured_frame_s, capture_device_s
Enumerator
rgb_555

16 bits per pixel: 5 bits for red, 5 bits for green, 5 bits for blue, and 1 bit of padding. No alpha.

rgb_565

16 bits per pixel: 5 bits for red, 6 bits for green, and 5 bits for blue. No alpha.

rgb_888

32 bits per pixel: 8 bits for red, 8 bits for green, 8 bits for blue, and 8 bits of padding. No alpha.

Function Documentation

◆ kc_capture_device()

capture_device_s& kc_capture_device ( void )

Returns a reference to the currently-active capture device interface. This interface allows the caller to communicate with the capture device.

Note
Calling this function will trigger an assertion failure if the device interface isn't available (e.g. because kc_initialize_capture() hasn't been called).
Warning
Calling this function before having called kc_initialize_capture() will trigger an assertion failure.
See also
kc_initialize_capture(), kc_release_capture()

◆ kc_force_input_resolution()

bool kc_force_input_resolution ( const resolution_s & r)

Asks the capture device to set its input resolution to the one given, overriding the current input resolution.

Note
If the resolution of the captured signal doesn't match this resolution, the captured image may become corrupted until the proper resolution is set.
Warning
This function may become obsolete in a future version of VCS, in which case its functionality will likely be merged into the capture device interface (see kc_capture_device()).

◆ kc_initialize_capture()

void kc_initialize_capture ( void )

Initializes the capture subsystem, including the capture hardware.

By default, VCS will call this function on program startup.

Note
Will trigger an assertion failure if the initialization fails.
See also
kc_release_capture()

◆ kc_release_capture()

void kc_release_capture ( void )

Releases the capture subsystem, including the capture hardware.

By default, VCS will call this function on program exit.

Returns true on success; false otherwise.

Warning
Calling this function – regardless of its return value – will invalidate the capture device interface reference returned by kc_capture_device().
